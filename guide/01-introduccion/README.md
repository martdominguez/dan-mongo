# Modulo 01: Introduccion a NoSQL y MongoDB

## Objetivo

Entender que es MongoDB, que significa NoSQL en la practica y en que casos tiene sentido usar una base de datos orientada a documentos en un backend.

## Que vas a aprender

- Que es MongoDB y que problema resuelve.
- Que significa NoSQL mas alla del nombre.
- En que se diferencia una base de datos relacional de una base de datos orientada a documentos.
- Que es BSON y por que importa.
- Que son una base de datos, una coleccion y un documento en MongoDB.
- En que escenarios reales de backend MongoDB suele funcionar bien.
- Cuando conviene usar MongoDB y cuando no.

## Enfoque del modulo

Este modulo es conceptual, pero no es abstracto. Cada idea se conecta con decisiones reales de backend: modelado de usuarios, pedidos, catalogos y respuestas de APIs.

Todavia no vamos a integrar MongoDB con Spring Boot. Eso aparece mas adelante. En este modulo nos enfocamos en entender bien el terreno para que despues las decisiones tecnicas tengan sentido.

## Archivos del modulo

- `theory.md`: conceptos clave y comparaciones.
- `examples.md`: ejemplos guiados con documentos y consultas simples.
- `exercises.md`: ejercicios breves orientados a practica.
- `quiz.md`: preguntas de repaso conceptual y aplicado.

## Resultado esperado

Al terminar este modulo deberias poder explicar con tus palabras:

- que es MongoDB
- por que NoSQL no significa "sin estructura"
- como pensar documentos y colecciones
- que trade-offs aparecen frente a una base de datos relacional
- en que tipo de backend lo elegirias y en cual preferirias otra opcion
