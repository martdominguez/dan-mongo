1. Que es MongoDB y cual es su unidad principal de almacenamiento.

2. Que significa NoSQL en la practica y por que no conviene interpretarlo como "sin estructura".

4. Que es BSON y por que resulta util en un backend real.

5. Cual es la diferencia entre base de datos, coleccion y documento en MongoDB.

6. Un pedido con items, direccion y estado que casi siempre se consulta completo, encaja mejor para pensar en tablas muy separadas o en un documento. Justifica brevemente.

7. Menciona dos casos de uso reales de backend donde MongoDB puede funcionar bien.

8. Menciona un caso donde una base de datos relacional probablemente sea mejor opcion que MongoDB.

9. Por que es un error pensar que usar MongoDB es solo "guardar JSON".

10. Si una API necesita devolver perfiles de usuario con datos variables y configuraciones opcionales, que ventaja podria aportar MongoDB.
