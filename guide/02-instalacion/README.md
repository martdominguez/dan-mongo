# Modulo 02: Instalacion, shell y herramientas basicas

## Objetivo

Preparar un entorno local simple para trabajar con MongoDB y aprender las primeras interacciones desde `mongosh`.

## Que vas a aprender

- Que piezas forman un entorno basico de MongoDB para desarrollo local.
- Como instalar MongoDB Community Edition de forma practica.
- Para que sirve `mongosh` y por que importa para un desarrollador backend.
- Cuando puede ayudar MongoDB Compass y por que es una herramienta opcional.
- Como conectarte a una instancia local y moverte entre bases y colecciones.
- Como crear una base de datos y una coleccion en los primeros minutos de trabajo.
- Como verificar que el entorno responde antes de integrar MongoDB con Spring Boot.

## Enfoque del modulo

Este modulo es practico desde el inicio. La idea no es memorizar herramientas, sino entender para que sirve cada una en el flujo real de un backend.

Todavia no vamos a profundizar en CRUD, modelado, indices ni rendimiento. Primero necesitamos un entorno funcional y una forma confiable de inspeccionar la base, probar comandos y validar que todo responde como esperamos.

## Archivos del modulo

- `theory.md`: entorno, herramientas y criterios de uso.
- `examples.md`: ejemplos guiados y ejecutables en `mongosh`.
- `exercises.md`: ejercicios breves para practicar el arranque del entorno.
- `quiz.md`: preguntas de repaso conceptual y aplicado.

## Resultado esperado

Al terminar este modulo deberias poder:

- instalar o dejar operativo un entorno local de MongoDB
- abrir `mongosh` y conectarte a una instancia
- ubicar la diferencia entre servidor, shell y herramienta grafica
- crear una base de datos y una coleccion
- ejecutar interacciones iniciales para comprobar que el entorno esta listo para trabajo backend
