# Quiz

## Preguntas de repaso

1. Cual es la diferencia practica entre `insertOne` e `insertMany`.

2. Cuando conviene usar `findOne` en lugar de `find`.

3. Que tipo de necesidad backend suele resolverse con un filtro como `{ active: true }`.

4. Que operadores basicos puedes usar para comparar valores numericos en MongoDB.

5. Para que sirve una proyeccion simple y que ventaja puede aportar en una API.

6. Que detalle importante hay que recordar sobre `_id` cuando haces una proyeccion inclusiva.

7. Que hace `sort({ createdAt: -1 })` en una consulta de pedidos.

8. Para que sirve `limit(5)` y por que puede ser util en un listado backend.

9. Cual es la diferencia de alcance entre `updateOne` y `updateMany`.

10. Cual es la diferencia de alcance entre `deleteOne` y `deleteMany`.

11. Si una API necesita devolver productos activos de una categoria, que tipo de operacion y que elementos de consulta esperarias usar.

12. Menciona un caso real donde una actualizacion puntual de una orden tenga sentido en backend.

13. Menciona un caso donde `deleteMany` podria ser util y otro donde habria que usarlo con mucho cuidado.

14. Por que este modulo prepara el terreno para trabajar luego con Spring Boot y `MongoRepository`.

## Autoevaluacion breve

- Puedo traducir una necesidad simple de una API a una operacion CRUD en MongoDB.
